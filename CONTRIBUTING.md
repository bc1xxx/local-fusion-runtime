# Contributing to Local Fusion Runtime

Thanks for your interest in contributing! This project aims to be a welcoming, low-friction open-source project.

## Getting Started

1. **Fork** the repository on GitHub.
2. **Clone** your fork:
   ```bash
   git clone https://github.com/your-username/local-fusion-runtime.git
   cd local-fusion-runtime
   ```
3. **Create a branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Install dependencies**:
   ```bash
   pip install -r requirements.txt
   ```
5. **Make your changes**.
6. **Run tests**:
   ```bash
   python -m pytest tests/ -v
   ```
7. **Commit** with a clear message:
   ```bash
   git commit -m "Add feature: your feature description"
   ```
8. **Push** to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
9. **Open a pull request** on GitHub.

## Guidelines

- Keep code simple and readable.
- Add type hints to new functions.
- Write tests for new functionality.
- Update docs if you change behavior.
- No cloud dependencies, no API keys, no telemetry.
- One PR per feature or bug fix.
